# 게임 기획 문서

## 0. 개발 목표
**데모 버전 완성이 1차 목표.** 모든 기획 요소를 완성할 필요 없음 — 핵심 루프(단어 획득 → 무기 조합 → 전투)가 끝까지 도는 플레이 가능한 데모에 집중.

## 1. 게임 한 줄 소개
단어를 모아 덱을 만들고, 그 단어로 무기를 조합해 전투하는 보글(Boggle) × Wuppo 스타일 보스전 액션 게임.

## 2. 핵심 컨셉
- **단어 습득 시스템**: Boggle 방식 기반. NPC와 대화 후, 특정 대화 페이지에서 Boggle 창을 띄워주고 그 안에서 단어를 찾아 획득.
- **전투 시스템**: Wuppo 스타일(보스전 위주의 2D 액션). 획득한 단어들을 전투 중 핫바에서 조합해 무기를 생성하고, 그 무기로 전투.
- **두 시스템의 연결**: 단어 획득(Boggle) → 무기 제작(조합) → 전투(무기 사용)으로 이어지는 구조. 단어의 동사 속성에 따라 만들어지는 무기 종류가 달라짐.
- **동사 구현 범위(데모 기준)**: **Hit, Throw, Shoot 구현**. **Place, Eat은 기획만 존재, 코드 구현은 데모 이후로 보류.**

## 3. 핵심 루프 (Core Loop)
1. NPC와 대화
2. 특정 대화 페이지에서 Boggle 창 오픈
3. Boggle에서 단어 탐색 및 획득 (이미 보유한 단어는 ✅ 표시, 모든 단어 획득 시 표시)
4. 전투 중 TAB키로 핫바 오픈 → 단어 조합 슬롯에 드래그앤드롭으로 단어 배치
5. 핫바 닫을 때 조합된 단어들의 어순([동사]+[형용사]+[명사]) 검증 → 통과 시 무기 생성
6. 생성한 무기로 전투 진행 (좌클릭 사용, 조합 단어 수에 따라 사용 횟수 차등)
7. [채우기] - 보스 처치 후 흐름 (보상, 다음 보스/스테이지 이동 등 — 데모에서는 단순 처치 확인 정도로 축소 가능)

## 4. 단어 → 무기 제작 시스템

**단어 보관 구조 (DeckManager)**
- 인벤토리 / 핫바 / 쿨타임 3단 구조
- 동사 1개 보장, 부족 시 강제 보충
- 리롤 가능

**무기 조합 방식**
- TAB 핫바 오픈(슬로우 모드) → 드래그앤드롭 조합 → 어순 검증 → 무기 생성
- 사용 횟수: 1단어=10회 ~ 5단어 이상=1회

**동사 타입(Verb Type) = 무기 종류 결정 (데모 구현 범위)**
- **Hit** (구현됨): 근접 타격. 단독 사용 가능, 명사 최대 1개.
- **Throw** (구현됨): 투척. 포물선 이동, 명사 여러 개 시 순차 연속 투척.
- **Shoot** (구현 예정): 직선 투사체로 예상. 명사 여러 개 시 순차 연속 발사. Throw와 유사한 구조이되 포물선 대신 직선 이동, 중력 영향 없음으로 예상.
- **Place** (기획만, 구현 보류): 설치형. 데모에서는 미구현, enum/데이터 구조만 유지.
- **Eat** (기획만, 구현 보류): 자가 효과형. 데모에서는 미구현, enum/데이터 구조만 유지.

**형용사(Adjective)**: Burn, Explosive 우선 구현 대상
- **Burn**: [채우기 - 효과, 지속시간, 데미지 방식]
- **Explosive**: [채우기 - 효과, 트리거 조건]

**덱 관리 시스템과의 관계**: DeckManager가 단어(동사/형용사/명사) 보관을 담당, 핫바에서 조합된 단어 조합이 무기로 변환되어 전투에서 소모됨.

## 8. 진행 구조 (스테이지/맵)
**아트 에셋(맵, 스프라이트, 타일셋 등)은 별도로 제작해서 가져올 예정.** 코드 측에서 해당 에셋을 적용할 수 있는 구조만 만들어두면 됨 (예: SpriteRenderer 슬롯, Animator 연동, 맵 오브젝트 배치를 위한 프리팹 구조 등). 맵/스테이지의 실제 디자인 내용은 기획 범위 밖.

## 9. 사운드/BGM 방향
**사운드/BGM 에셋도 별도로 제작해서 가져올 예정.** AudioSource 연동 등 재생 가능한 코드 구조만 준비. 구체적 톤/곡 계획은 기획 범위 밖.

## 10. 애니메이션 시스템 요구사항
플레이어와 보스 모두 **상태 변화에 따라 애니메이션이 자동 전환되는 구조** 필요:
- **플레이어**: 대기(Idle), 이동(Move), 점프(Jump), 공격(Attack, 동사별로 다를 수 있음), 피격(Hurt), 사망(Death) 등 상태별 애니메이션 트리거
- **보스**: 대기(Idle), 이동(Move, 해당하는 경우), 각 패턴별 모션, 피격(Hurt), 사망(Death) 상태별 애니메이션 트리거
- 현재는 스프라이트가 placeholder(사각형)이고 일부 동작은 직접 transform 조작으로 구현되어 있음 (예: HitWeaponObject의 Animator는 이미 사용 중, 본체 캐릭터는 아직 Animator 미적용 추정)
- **요청 사항**: Animator Controller + 상태별 트리거/파라미터를 코드에서 호출하는 구조로 구현. 실제 애니메이션 클립/스프라이트는 나중에 끼워 넣을 수 있도록 빈 슬롯 형태로 준비.
